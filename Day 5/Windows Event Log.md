# Day 5 – Windows Event Logs (Learning)

## Objective

Learn how Windows records system activities using Event Logs and understand important Security Event IDs commonly monitored by SOC analysts.

---

# What are Windows Event Logs?

Windows Event Logs are records of activities occurring on a Windows system.

These logs help administrators and security analysts:

- Monitor system health
- Detect suspicious activity
- Troubleshoot problems
- Investigate security incidents
- Perform forensic analysis

Windows continuously records events generated by:

- Users
- Applications
- Services
- Drivers
- Operating System
- Security Components

---

# Event Viewer

Event Viewer is a built-in Windows tool used to view and analyze system logs.

Open Event Viewer:

```
Win + R
```

Type:

```
eventvwr.msc
```

Or

```
Start → Event Viewer
```

---

# Windows Log Structure

```
Event Viewer

├── Custom Views
├── Windows Logs
│     ├── Application
│     ├── Security
│     ├── Setup
│     ├── System
│     └── Forwarded Events
│
└── Applications and Services Logs
```

---

# Windows Log Types

## 1. Security Log

Contains security-related events.

Examples:

- User Logins
- Failed Login Attempts
- User Creation
- Group Changes
- Privilege Assignments
- Process Creation
- Account Lockouts

Used by:

- SOC Analysts
- Incident Responders
- Threat Hunters

---

## 2. System Log

Contains events generated by Windows and hardware.

Examples:

- Driver Loading
- Service Start/Stop
- System Shutdown
- System Startup
- Hardware Errors

Generated by:

- Windows Kernel
- Device Drivers
- Services

---

## 3. Application Log

Contains events generated by installed applications.

Examples:

- Microsoft Office
- SQL Server
- Chrome
- IIS
- Custom Applications

Useful for troubleshooting application failures.

# Important Windows Event IDs

---

# Event ID 4624 – Successful Logon

Indicates that a user successfully authenticated.

Log Location:

```
Windows Logs
    └── Security
```

Important Fields:

- Account Name
- Security ID
- Logon Type
- Authentication Package
- Workstation Name
- Source Network Address

SOC Use Cases:

- Successful login tracking
- User activity monitoring
- Lateral movement detection
- Login timeline analysis

---

# Event ID 4625 – Failed Logon

Generated when authentication fails.

Common Reasons:

- Wrong Password
- Invalid Username
- Locked Account
- Disabled Account

Important Fields:

- Account Name
- Failure Reason
- Status Code
- Logon Type
- Source IP

SOC Use Cases:

- Brute Force Detection
- Password Spray Detection
- Unauthorized Access Attempts

---

# Event ID 4688 – Process Creation

Generated whenever a new process starts.

Important Fields:

- Process Name
- Command Line
- Parent Process
- User
- Process ID

Example:

```
powershell.exe

cmd.exe

notepad.exe
```

SOC Use Cases:

- Detect PowerShell abuse
- Detect Command Prompt execution
- Malware detection
- Suspicious process execution

---

# Event ID 4672 – Special Privileges Assigned

Generated when an account logs in with administrative privileges.

Usually seen for:

- Administrator
- SYSTEM
- Service Accounts

Privileges include:

- SeDebugPrivilege
- SeBackupPrivilege
- SeRestorePrivilege

SOC Use Cases:

- Privileged account monitoring
- Privilege abuse detection
- Administrator login tracking

---

# Event ID 4720 – User Account Created

Generated when a new local or domain user account is created.

Important Fields:

- New Username
- SID
- Creator Account

SOC Use Cases:

- Detect unauthorized account creation
- Insider threat investigations
- Persistence detection

---

# Event ID 4732 – User Added to Local Security Group

Generated when a user is added to a security-enabled local group.

Example:

```
Administrators

Remote Desktop Users
```

Important Fields:

- User Added
- Group Name
- Performed By

SOC Use Cases:

- Privilege escalation detection
- Administrator group monitoring
- Unauthorized group modifications

---

# Event ID 7045 – New Service Installed

Located in:

```
Windows Logs
    └── System
```

Generated when a new Windows service is installed.

Important Fields:

- Service Name
- Executable Path
- User
- Service Type

SOC Use Cases:

- Malware persistence
- Suspicious service installation
- Ransomware investigations

# Why SOC Analysts Monitor These Events

These Event IDs help detect:

- Brute Force Attacks
- Privilege Escalation
- Malware Execution
- Persistence Techniques
- Insider Threats
- Unauthorized User Creation
- Suspicious Service Installation
- Lateral Movement

These events are commonly monitored in SIEM solutions such as Microsoft Sentinel, Splunk, QRadar, ArcSight, and Elastic Security.

# Key Takeaways

- Windows Event Logs record activities performed by users, applications, services, and the operating system.
- Event Viewer is the primary tool for viewing and analyzing Windows logs.
- The three main log categories are **Security**, **System**, and **Application**.
- Event IDs uniquely identify specific activities, making them essential for security monitoring and investigations.
- Understanding common Event IDs is a foundational skill for SOC analysts, threat hunters, and incident responders.
