# Day 8 – Windows Security Features Lab

## Objective

Gain hands-on experience with Windows built-in security features by configuring Microsoft Defender Antivirus, Windows Defender Firewall, Task Scheduler, and PowerShell. Learn how to review security logs generated by these components.

---

# Lab 1 – Microsoft Defender Antivirus

## Objective

Inspect Microsoft Defender settings and perform a quick scan.

### Steps

1. Open **Windows Security**.

```
Settings
    → Privacy & Security
        → Windows Security
```

or press:

```
Win + R
```

Type:

```
windowsdefender:
```

2. Open **Virus & Threat Protection**.

3. Verify:

- Real-time Protection
- Cloud-delivered Protection
- Automatic Sample Submission
- Tamper Protection

4. Run a **Quick Scan**.

---

## View Protection History

Navigate to:

```
Windows Security
    → Virus & Threat Protection
        → Protection History
```

Observe:

- Detected threats
- Quarantined files
- Allowed threats
- Scan history

---

## SOC Investigation

Questions to ask:

- Was malware detected?
- Was the file quarantined?
- Which user executed the file?
- What was the file path?

# Lab 2 – Windows Defender Firewall

## Objective

Create and verify a custom firewall rule.

### Open Firewall

Press:

```
Win + R
```

Type:

```
wf.msc
```

---

## Create an Inbound Rule

1. Select **Inbound Rules**.
2. Click **New Rule**.
3. Choose **Port**.
4. Select **TCP**.
5. Specify port:

```
8080
```

6. Choose:

```
Block the connection
```

7. Apply to:

- Domain
- Private
- Public

8. Name the rule:

```
Block Port 8080 Lab
```

Finish the wizard.

---

## Verify Rule

Locate the newly created rule under **Inbound Rules**.

---

## SOC Investigation

Review:

- Rule name
- Direction
- Action (Allow/Block)
- Enabled status
- Profiles applied

---

# Lab 3 – Task Scheduler

## Objective

Create a scheduled task that opens Notepad at user logon.

### Open Task Scheduler

```
taskschd.msc
```

---

### Create Task

1. Select **Create Basic Task**.
2. Name:

```
Open Notepad Lab
```

3. Trigger:

```
When I log on
```

4. Action:

```
Start a Program
```

Program:

```
C:\Windows\System32\notepad.exe
```

5. Finish the wizard.

---

## Verify

Sign out and sign back in.

Notepad should open automatically.

Delete the task after testing if no longer needed.

---

## SOC Investigation

Review:

- Task name
- Trigger
- Action
- Author
- Last Run Time

# Lab 5 – Review Windows Defender Logs

## Objective

Review Windows Defender operational logs.

Open:

```
eventvwr.msc
```

Navigate to:

```
Applications and Services Logs
    └── Microsoft
        └── Windows
            └── Windows Defender
                └── Operational
```

Observe events related to:

- Scan started
- Scan completed
- Threat detected
- Threat quarantined
- Signature updates

# Key Takeaways

- Microsoft Defender Antivirus provides built-in malware protection and records security events that can be reviewed in Event Viewer.
- Windows Defender Firewall uses rules to allow or block network traffic and should be configured according to organizational security policies.
- Task Scheduler automates tasks and should be monitored because it can be used for both legitimate administration and persistence.
- PowerShell is a powerful administrative tool that provides detailed visibility into Windows system configuration.
- Reviewing Defender and Firewall logs helps SOC analysts identify malware detections, configuration changes, and suspicious activity during incident investigations.
